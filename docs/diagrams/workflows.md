## 01. The merge workflow {#d-01}

- **Kind**: `sequence`
- **Source page**: [quality/clients/duplicate-merge](/docs/quality/clients/duplicate-merge)
- **Originating section**: The merge workflow

```mermaid
sequenceDiagram
    autonumber
    participant U as Operator
    participant Page as Duplicates screen
    participant DB as DB
    participant FS as Backup files

    Page->>DB: Detect candidates
    Page-->>U: List of candidate pairs
    U->>Page: Pick (source, receiver) for each merge
    U->>Page: Confirm merge
    loop each linked table
        Page->>DB: Move source's rows to receiver<br/>(SUM where appropriate, e.g., ClientFinans balance)
    end
    Page->>FS: Write backup files /upload/dublicate/{id}-...-{ts}.txt
    Page->>DB: DELETE source Client row
    Page->>DB: TRUNCATE {{cache}} (clear caches)
    Page-->>U: Merge complete
```

## 03. The workflow {#d-03}

- **Kind**: `sequence`
- **Source page**: [quality/clients/create-edit-client](/docs/quality/clients/create-edit-client)
- **Originating section**: The workflow

```mermaid
sequenceDiagram
    autonumber
    participant U as Operator
    participant F as Form
    participant DB as Client DB
    participant Side as Visiting / VisitExp / Tags / Phones
    participant Agt as Agents-packet cache

    U->>F: Fill form (name, category, region, city, type, channel, …)
    U->>F: Save
    F->>DB: Validate required (NAME, CLIENT_CAT, REGION, CITY)
    alt invalid
        F-->>U: Field errors
    else
        F->>DB: INSERT Client (ACTIVE='Y', CREATE_BY, CREATE_AT)
        F->>Side: INSERT ClientPhones (up to 2)
        F->>Side: INSERT Visiting rows (one per agent × weekday)
        F->>Side: INSERT VisitExp if expeditor set
        F->>Side: INSERT SalesCategory rows + ModelTag rows
        opt contragent mode
            F->>DB: INSERT Contragent parallel record
        end
        F->>Agt: AgentPaket::enableFullRefreshByClient()<br/>(invalidate every linked agent's cache)
        F-->>U: Saved
    end
```

```mermaid
sequenceDiagram
    autonumber
    participant U as Admin
    participant Form as User form
    participant Lic as Licence check
    participant DB as User table
    participant RBAC as Permission grid
    participant Cash as Cashbox catalogue

    U->>Form: Open Users → New

        Form->>DB: Phone unique?

            Form-->>U: "Этот телефон уже есть в базе"

                Form-->>U: License-blocked error
            else within cap
                Form->>DB: INSERT user (role, active, password hash)
                Form->>RBAC: Apply default role grants
                opt role = cashier
                    Form->>Cash: Bind selected cashboxes to this user
                end
                Form-->>U: Success
            end
        end
    end

sequenceDiagram
    autonumber
    participant Ex as Expeditor (mobile)
    participant PD as PaymentDeliver table
    participant Mgr as Manager (Approval screen)
    participant CT as ClientTransaction
    participant Inv as Invoice row (TRANS_TYPE=1)
    participant CF as ClientFinans cache
    participant Closed as Closed-period gate
    participant Notif as Notification

    Ex->>PD: Submit payment from field (CONFIRM=0)

        Mgr->>PD: Set CONFIRM=1, write APPROVED_BY
        Mgr->>CT: INSERT TRANS_TYPE=3 receipt, SUMMA=approved amount, CONFIRM_ID=PaymentDeliver.ID
        alt invoice row exists for this order
            Mgr->>Inv: Update COMPUTATION (apply payment to invoice)
            Note over Inv: ⚠ The new receipt is linked back<br/>to the invoice via TransactionClosed.
        end
        Mgr->>CF: correct() — recompute running balance
        Mgr->>Notif: paymentAcceptedNotification — push/SMS to client
        Mgr-->>Mgr: Refresh — row disappears (CONFIRM=1 hidden)
    end
```

    U->>Form: Build order
    opt user picks a per-line discount rule
        U->>Form: Pick discount on line X
        Form->>Skidka: Is this rule active? Is this agent allowed?
        alt rule invalid
            Form-->>U: Field error
        else
            Form->>Form: Apply the rule's percentage to that line's total
        end
    end
    opt user types a per-line amount (price-type permitting)
        U->>Form: Type per-unit discount amount on line Y
        Form->>Form: Multiply by line quantity → stored as line discount
    end
    U->>Form: Press Save
    Form->>Sales: Save order + lines (line discounts inline)
    Note over Sales,Skidka: ⚠ Auto-discount runs AFTER save.<br/>The order is already written; this adds<br/>a link record between the order and the rule.
    Sales->>Skidka: Find any auto-discount rule that matches the order
    alt match found
        Skidka->>Sales: Insert link record between order and rule
        Sales->>Sales: Update the auto-discount-related totals
    else no match
        Skidka-->>Sales: Nothing happens
    end
    Sales-->>U: Saved (with whatever combination of discount paths applied)
