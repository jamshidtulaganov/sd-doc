# sd-main — CRM дилера

**sd-main** (внутреннее название *Novus Distribution*) — CRM каждого дилера в сердце платформы SalesDoctor. У каждого дилера свой инстанс `sd-main` со своей MySQL-схемой (префикс `d0_`). Он собирает заказы, управляет полевой командой, управляет товарными остатками и складами, проводит платежи, интегрируется с 1C / Didox / Faktura.uz и предоставляет мобильный + B2B-онлайн каналы.

Поддеревья ниже разбивают документацию sd-main тремя способами. Выберите то, что соответствует вашей задаче.

## С первого взгляда

| Свойство | Значение |
|----------|-------|
| Стек | PHP 7.3 · Yii 1.x · Nginx + php-fpm · MySQL 8 · Redis 7 |
| Тенантность | DB-per-customer, маршрутизация по поддомену через `TenantContext` |
| Mobile | Поверхность api3 (Bearer-token аутентификация) для приложения агента |
| B2B online | Поверхность api4 (token + Telegram-signed init) |
| Модули | 40+ Yii-модулей под `protected/modules/` |
| Диаграммы | Inline Mermaid в исходных доках; канонический FigJam в [галерее диаграмм](../diagrams/index.md) |

## Как sd-main соотносится с другими двумя проектами

- **`sd-billing`** находится выше по течению — он пушит файлы лицензий и обновления справочника телефонов в каждый sd-main и выставляет дилерам счёт за платформу. См. [Карту межпроектной интеграции](../ecosystem.md#inter-project-integration-map).
- **`sd-cs`** находится ниже по течению — он открывает read-only соединения в базу каждого sd-main, чтобы строить консолидированные отчёты HQ-уровня. См. [интеграция sd-cs ↔ sd-main](../sd-cs/sd-main-integration.md).

Поверхность межпроектного взаимодействия маленькая и однонаправленная: sd-main экспонирует узкий набор эндпоинтов `api/billing` и `api/license`; всё остальное скоупится одним дилером.
