# Didox EDI

Didox — оператор электронных документов, используемый многими узбекскими
B2B-клиентами. SalesDoctor отправляет электронные счета-фактуры,
договоры и акты в Didox; контрагент подписывает / отклоняет в Didox,
и мы зеркалим статус.

## Что мы отправляем

| Тип документа | Триггер |
|---------------|---------|
| Электронный счёт-фактура | Заказ достигает `Loaded` |
| Акт выполненных работ | Заказ достигает `Delivered` |
| Договор | Вручную из модуля договоров |

## Аутентификация

OAuth2 client credentials. Токены кешируются в `redis_app` на 1 час.

## Отслеживание

Каждый отправленный документ получает Didox-ID, сохраняемый обратно на
заказе / договоре. `IntegrationLog` хранит сетевые пейлоады.

## Режимы отказа

- ИНН контрагента отсутствует в `Client.INN` → `MAPPING_MISSING`.
- HTTP 5xx от Didox → повтор с задержкой.
- Контрагент отклоняет в Didox → статус зеркалится обратно в SD;
  отображается как уведомление оператору.
