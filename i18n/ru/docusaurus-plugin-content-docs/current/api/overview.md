# Обзор API

Существует **четыре версионированных поверхности API**, каждая из которых является отдельным модулем Yii:

| Версия | Модуль | Аудитория | Статус |
|--------|--------|-----------|--------|
| **v1** | `api` | Устаревшие server-to-server интеграции, партнёрские интеграции | Поддержка |
| **v2** | `api2` | Старые мобильные клиенты | Заморожен |
| **v3** | `api3` | **Текущее мобильное приложение агента** | Активен |
| **v4** | `api4` | **B2B / портал онлайн-заказов / новые мобильные клиенты** | Активен |

URL соответствуют шаблону `/<module>/<controller>/<action>`. Примеры:

- `POST /api3/login/index`
- `GET  /api3/order/list`
- `POST /api4/order/create`
- `POST /api/json1c/import`

## Соглашения

- **Транспорт**: только HTTPS (TLS терминируется на Nginx).
- **Аутентификация**: на основе токенов (`api3` / `api4`); общий секрет или
  basic auth для некоторых эндпоинтов v1. См. [Аутентификация](./authentication.md).
- **Формат**: JSON в запросах/ответах. В v1 есть несколько XML-эндпоинтов
  (`xml1c`, `pradata`).
- **Дата/время**: ISO 8601 в UTC, если эндпоинт не оговаривает иное.
- **Ошибки**: HTTP-статус + JSON-тело
  ```json
  { "success": false, "error": "Описание ошибки", "code": "INVALID_TOKEN" }
  ```
- **Пагинация**: query-параметры `?page=N&limit=M`. Ответ включает
  `total`, `page`, `limit`.
- **Локаль**: `?lang=ru|en|uz` переопределяет локаль пользователя по умолчанию.
- **Rate limit**: per-IP на шлюзе. Application-level rate limit на эндпоинтах
  аутентификации (`LoginController` ограничивает неуспешные попытки).

## Политика версионирования

- **v3 и v4 стабильны**. Ломающие изменения выходят как новые эндпоинты,
  никогда не in-place.
- **v1 и v2 заморожены** — файлы `*.obsolete` в их контроллерах — это
  удалённый код, оставленный для археологии.
