# ADR 0004 — Версионированные API-модули (api / api2 / api3 / api4)

- Status: **Accepted**
- Date: 2024-08-08
- Deciders: API working group

## Контекст

Мобильные клиенты нельзя force-upgrade. Мобильная сборка v1 от 2018-го
всё ещё на устройствах в полях. Ломать контракт — не вариант.

## Решение

Каждое поколение API живёт в **собственном Yii-модуле** под
`protected/modules/api*`. Новые эндпоинты идут в последнюю активную версию
(сейчас v3 для мобильного, v4 для онлайн / B2B). v1 / v2 заморожены.

## Последствия

- ✅ Старые клиенты продолжают работать без изменений.
- ✅ Инженеры с одного взгляда видят, какую поверхность они трогают.
- ❌ Дублирование кода: схожие эндпоинты есть в v3 и v4. Митигируется
  выносом логики в общие сервисы.
- ❌ Discoverability для внешних потребителей — решается этим
  сайтом документации.
