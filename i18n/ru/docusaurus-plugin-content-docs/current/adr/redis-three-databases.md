# ADR 0003 — Redis: три логические базы данных

- Status: **Accepted**
- Date: 2024-02-20
- Deciders: Platform team

## Контекст

Один инстанс Redis использовался для сессий, очереди и кеша приложения,
шарили один keyspace. `FLUSHDB` для очистки устаревшего кеша
убивал сессии. Метрики глубины очереди загрязнялись cache-ключами.

## Решение

Использовать **три логические базы Redis** на одном инстансе:

| DB | Назначение |
|----|---------|
| 0 | Сессии (`keyPrefix` = `HTTP_HOST`) |
| 1 | Очередь |
| 2 | Кеш приложения (скоупится через `TenantContext`) |

`FLUSHDB` на одной больше не влияет на остальные.

## Последствия

- ✅ Чище разделение; каждый компонент владеет своей БД.
- ✅ Раздельные метрики памяти на каждую заботу.
- ❌ Чуть больше церемонии в конфиге на окружение.
- ❌ Программисты должны помнить — **никогда** не использовать
  `Yii::app()->cache` напрямую — только через `ScopedCache`.
