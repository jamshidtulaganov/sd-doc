# ADR 0001 — Остаться на Yii 1.x пока

- Status: **Accepted**
- Date: 2024-09-12
- Deciders: Engineering leads

## Контекст

Приложение — это 12+ летняя кодовая база на Yii 1.x с массовым
использованием `CActiveRecord`, magic-геттеров и набором вендорных
расширений. PHP 7.3 — end-of-life. Yii 1.x официально без поддержки, но
community-форк (`yii1/yii1-php8`) поставляет фиксы для PHP 8.

Мы рассматривали:

1. **Остаться на Yii 1.x + PHP 7.3** (статус-кво).
2. **Yii 1.x + PHP 8.2** через community-форк.
3. **Миграция на Yii 2 / Yii 3.**
4. **Миграция на Laravel.**
5. **Strangler fig: переписывать кусок за куском в новый фреймворк.**

## Решение

Остаться на **Yii 1.x** на обозримое будущее. Запланировать апгрейд PHP 8.2
за флагом, как только community-форк пройдёт нашу test matrix.

## Последствия

- ✅ Нулевая стоимость переписывания в краткосрочной перспективе.
- ✅ Существующие навыки команды передаются 1:1.
- ❌ Должны продолжать гонять EOL PHP, пока не приземлится апгрейд форка.
- ❌ Не сможем легко привлекать новых нанимаемых, ожидающих Yii 2 / Laravel.
- ❌ Некоторые современные фичи (typed properties, attributes) недоступны.

## Митигации

- WAF перед приложением.
- Агрессивный цикл патчинга контейнеров.
- Новые фичи должны лежать на границе strangler, чтобы можно было пересмотреть
  выбор фреймворка для следующего major.
