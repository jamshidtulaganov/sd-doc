# Масштабирование

## Сначала вертикально

Yii 1 — щадящий; одна машина 8 ядер / 16 ГБ комфортно тащит mid-size
тенантов. Выжмите больше из одной машины, прежде чем идти горизонтально:

- `pm = dynamic` для php-fpm с `pm.max_children` ≈ `(memory_avail /
  avg_php_mem)`.
- Тюньте `innodb_buffer_pool_size` до ~70 % RAM хоста БД.
- `redis maxmemory-policy = allkeys-lru` для cache db (db2).

## Горизонтально

App-уровень stateless → запускайте N реплик за LB. Sticky-сессии **не**
нужны (сессии в Redis).

Воркеры: запускайте отдельный набор реплик; каждая реплика берёт из db1.

База данных: сначала вертикально (read-реплики — опция, но JOIN-ы между
БД тенантов редки).

## Горячие таблицы

| Таблица | Митигация |
|-------|------------|
| `gps_track` | Дневное партицирование; truncate старше 90 дней |
| `integration_log` | Дневное партицирование; архив через 30 дней |
| `audit_*` | Партицирование на тенанта |
| `order` (большие тенанты) | Ревью индексов; рассмотрите `(STATUS, DATE)` |

## CDN

Поставьте CDN (Cloudfront, Bunny) перед `static/`, `assets/`, `js/`,
`css/` с длинным max-age. Папка `assets/` с хеш-именами делает
инвалидацию кеша автоматической.
