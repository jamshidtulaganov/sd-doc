# Вайрфреймы — текущий UI SalesDoctor

Этот раздел — для **новых фронтенд-разработчиков**, присоединяющихся к
команде. Он фиксирует, **как приложение выглядит сегодня**, чтобы вы
могли соответствовать существующим паттернам, когда строите новые
экраны.

Это намеренно справочник по **вайрфреймам / раскладке**, а не глубокая
дизайн-система — визуальный стиль (цвета, шрифты, микро-интеракции)
живёт в Figma-файле `SD-Web-old.fig`.

## Источник

Скриншоты страниц, на которые ссылается ниже, извлечены из Figma-файла
пользователя `SD-Web-old.fig`. Под `static/wireframes/extracted-from-figma/page-NN-WIDTHw.png`
лежат **60 вайрфреймов уровня страницы**.

Если хотите свежие экспорты — открывайте Figma-файл напрямую:

> `~/projects/salesdoctor/sd-docs/static/wireframes/SD-Web-old.fig`

(или загрузите его на figma.com, чтобы получить shareable URL).

## Канонический layout

Каждая страница в admin-приложении SalesDoctor разделяет этот скелет.

```
┌──────────────────────────────────────────────────────────────────────┐
│  TOP BAR                                                             │
│  Logo · role tabs (Sales / Касса / GPS / Онлайн-помощь)              │
│      · search · date range · balance · notifications · user         │
├──────┬───────────────────────────────────────────────────────────────┤
│ LEFT │   PAGE                                                        │
│ NAV  │   ┌─ breadcrumbs ────────────────────────────────────────┐    │
│      │   └──────────────────────────────────────────────────────┘    │
│ icon │   ┌─ page title + primary actions ──────────────────────┐    │
│ rail │   └──────────────────────────────────────────────────────┘    │
│      │   ┌─ filters ───────────────────────────────────────────┐    │
│  …   │   └──────────────────────────────────────────────────────┘    │
│      │   ┌─ content (table / form / map / cards) ──────────────┐    │
│      │   └──────────────────────────────────────────────────────┘    │
└──────┴───────────────────────────────────────────────────────────────┘
```

### Top bar

- Слева: логотип / бренд
- Слева в центре: role-driven вкладки (Sales, Касса, GPS, Онлайн-помощь, …)
- В центре: глобальный поиск «Найти страницы (Ctrl+K)»
- Справа: пикер диапазона дат, balance-пилюля, уведомления, меню пользователя

### Левый навигационный рейл

Вертикальный icon-рейл с подписями:

`Планы · Заявки · Склад · Клиенты · Агенты · Отчеты · Настройки ·
Аудит 2 · Команда · Диагностика`

Рейл **фильтруется по ролям** — разные роли видят разные пункты.

### Область страницы

Контентная область всегда содержит, в порядке:

1. **Breadcrumbs**
2. **Заголовок страницы** + **primary action кнопка** в правом верхнем углу
3. **Filter bar** (сверху) или **filter rail** (слева, для тяжёлых отчётов)
4. **Контент** — таблица, форма, карта+сайдбар, KPI-сетка или detail-панель

## Паттерны страниц (с примерами)

### 1. KPI-плитки дашборда

Большие цветные блоки с одной метрикой, plan vs fact ниже. Используется
на главном дашборде и KPI-экранах.

![KPI tiles](/wireframes/extracted-from-figma/page-0-1512w.png)

Применяйте: 4 плитки в ряду на десктопе, 2 в ряду на узком. Цвет —
threshold-driven (красный = ниже плана, зелёный = на / выше плана).

### 2. List + filters (по умолчанию)

Сверху filter-чипы → таблица → футер с пагинацией. Самый частый паттерн.

```
┌─────────────────────────────────────────────────────────────┐
│ [date▾] [agent▾] [status▾] [type▾] [+ more filters]   [Add]│
├─────────────────────────────────────────────────────────────┤
│  table                                                       │
└─────────────────────────────────────────────────────────────┘
```

### 3. Form + summary (детальное редактирование)

Слева = секции формы; справа = «Итоги» + связанная таблица.

![Stock transfer page](/wireframes/extracted-from-figma/page-1-3132w.png)

Выше — страница **перемещения товара (Перемещение товара)**:
- Левая колонка: форма (Со склада, На склад, ТСД Сканер, Комментарий).
- Правая колонка: live-сводка позиций + итогов.
- Ниже: вкладки (Все / Напитки / Шоколад / …) + таблица товаров.

### 4. Карта + side-панель

Full-bleed карта с пристёгнутой правой панелью. Используется для GPS,
мониторинга, планирования маршрутов.

![GPS monitoring](/wireframes/extracted-from-figma/page-13-2922w.png)

Правая панель переключается между режимами **Мониторинг** и **Маршрут**.

### 5. Master list + detail-панель

Для сущностей, которые хочется просматривать и нырять в них без смены
страниц. Две колонки: таблица слева, детали выбранной записи справа.

### 6. Вкладки над одной контентной областью

Используются внутри detail-страниц для разбиения длинных форм на
«секции без скролла» (например, Profile, Plans, Logs).

## Шпаргалка по компонентам

| Паттерн | Где в legacy Yii views | Замена, если есть |
|---------|------------------------|-------------------|
| Top bar + sidebar | `protected/views/layouts/main.php` (sd-main) | – |
| Ссылки сайдбара | `protected/views/partial/sidebar.php` | – |
| Filter bar | per-controller view, jQuery | – |
| DataTable | `protected/views/orders/list.php` и т.д. + `js_plugins/FixedColumns` | – |
| Modal | `protected/views/modals/*` + `js_plugins/fancybox2` | – |
| KPI-плитка | `protected/modules/dashboard/views/...` | – |
| Map | `protected/modules/gps3/views/...` + `ng-modules/gps/` | Angular-компонент |
| Charts | `js_plugins/jquery-highcharts-10.3.3` | – |

## Справочник извлечённых страниц

Все 60 страниц лежат под `/wireframes/extracted-from-figma/`. Они
названы `page-N-WIDTHw.png`. Откройте папку в редакторе, чтобы пройти
их все; рекомендуем:

```bash
open sd-docs/static/wireframes/extracted-from-figma/
```

Берите ближайшую существующую страницу при проектировании нового
экрана — соответствуйте её layout'у, плотности и выбору компонентов,
если нет конкретной причины отступать.

## Кладите свои экспорты

Добавляйте новые вайрфреймы экспортом нужного Figma-фрейма как PNG (2×)
и кладите в `static/wireframes/`. Ссылайтесь из markdown-страницы:

```markdown
![Outlet detail](/wireframes/outlet-detail.png)
```

Соглашение по неймингу: `<area>-<page>.png`, например, `orders-list.png`,
`client-detail.png`, `report-bonus.png`.
