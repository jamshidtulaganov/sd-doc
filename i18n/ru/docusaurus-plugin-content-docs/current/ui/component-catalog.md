# Каталог компонентов (на уровне вайрфреймов)

Это каталог компонентов **на уровне вайрфреймов**. Он перечисляет каждую
переиспользуемую UI-часть, которую вы увидите в админке, для чего она
используется и где живёт существующая реализация. Это **не** Storybook —
визуальный дизайн живёт в `SD-Web-old.fig`.

Когда строите что-то новое, найдите ближайшее совпадение здесь и
переиспользуйте его markup-паттерн.

## Layout-обвязка

### Top bar

```
┌──────────────────────────────────────────────────────────────┐
│  ☰  LOGO  | Sales · Касса · GPS · Онлайн-помощь | 🔍 search   │
│                       date-range · 💰 balance · 🔔 · 👤      │
└──────────────────────────────────────────────────────────────┘
```

**Где**: каждая страница. **Markup**:
`protected/views/layouts/main.php` → `views/partial/topbar.php`.

### Левый рейл (сайдбар)

```
🗒  Планы
📑  Заявки
📦  Склад
👥  Клиенты
🛵  Агенты
📊  Отчёты
⚙  Настройки
🔍  Аудит
👨‍👩‍👧  Команда
🩺  Диагностика
```

**Где**: каждая страница (кроме modal-only views). **Role-driven** —
RBAC скрывает пункты, к которым у пользователя нет доступа.

### Breadcrumbs

`Dashboard › Orders › List` — обязательны при глубине больше 2.

### Page header

```
Page Title                                       [+ New thing]
```

Primary-действие справа сверху. Secondary-действия в overflow `⋮`.

## Tabular-компоненты

### Дефолтная таблица данных

| col | col ↕ | col | … | ⋮ |

- Первая колонка = номер строки или чекбокс (multi-select)
- Header-строка sticky
- `↕` обозначает сортируемое
- Денежные колонки — выравнивание по правому краю
- Колонка статуса = цветная пилюля (см. Status pill ниже)
- Последняя колонка = `⋮` с действиями строки (Edit / Duplicate / Delete)
- Футер: пагинация + селектор размера страницы + общее количество

**Где**: каждый list-экран (заказы, клиенты, агенты, оплаты, …).
**Библиотека**: DataTables + `js_plugins/FixedColumns`.

### Сгруппированная / раскрывающаяся таблица

```
▶ Group A    sum=…    count=…
  ┌─ row …
  └─ row …
▶ Group B
```

**Где**: иерархические отчёты (per-agent → per-order, per-category
→ per-product).

### Inline-редактирование ячейки таблицы

Двойной клик по ячейке → она становится input → blur или Enter
сохраняют через AJAX.

**Где**: типы цен, категории, настройки агента.

## Tile / card-компоненты

### KPI-плитка

```
┌─ Title ───────────────────────────┐
│  1,247                            │
│  Plan: 161        Fact: 3 + 0     │
└───────────────────────────────────┘
```

- Большое число
- Тонкая строка plan / fact
- Цвет фона = threshold (красный / янтарный / зелёный)
- 4 в ряду на десктопе, 2 на ≤ 1024 px

**Где**: дашборд. **Вайрфрейм**: [page-0](/wireframes/extracted-from-figma/page-0-1512w.png).

### Detail-панель карточки

```
┌─ Card title ──────────────────────┐
│  Field 1:    value                │
│  Field 2:    value                │
│  …                                │
└───────────────────────────────────┘
```

**Где**: detail клиента, detail агента, summary заказа.

## Status-пилюли

| Фон | Значение |
|-----|----------|
| Зелёный | OK / Paid / Delivered / Active |
| Янтарный | Pending / Awaiting approval |
| Красный | Cancelled / Defect / Expired |
| Серый | Draft / Closed / Archived |
| Синий | In progress / Loaded |

Всегда включайте текстовый ярлык внутри пилюли — цвет вспомогательный,
никогда не единственный сигнал (доступность).

## Фильтры

### Top filter bar

```
[ Date range ▾ ] [ Status ▾ ] [ Agent ▾ ] [ + More filters ]   [ Apply ]
```

- ≤ 6 видимых чипов, остальные сворачиваются за «More filters».
- Дефолтная дата = последние 30 дней.
- Состояние фильтра сохраняется в URL query params (чтобы ссылками
  можно было делиться).

### Filter rail (слева, для тяжёлых отчётов)

```
▣  Date
   From [_]
   To   [_]
▣  Status
   ☑ New / ☐ Loaded / ☑ Paid
▣  Agent
   [Search ]
[Apply] [Reset]
```

**Используйте, когда**: ≥ 10 фильтров или нужна группировка.

## Формы

### Секционная форма

```
┌─ Section title ──────────────────────┐
│  Field   [_______________________]   │
│  Field   [______]   Toggle  ☑        │
└──────────────────────────────────────┘
```

- Две колонки на десктопе, одна на узком.
- Обязательные поля помечены красной звёздочкой.
- Inline-валидация по полю; баннер сверху для cross-field-ошибок.
- Sticky-футер для форм, длиннее viewport'а.
- Primary `Save` (справа), secondary `Cancel` (слева); плюс опциональный
  `Save and add another`.

### Inline-редактирование

Двойной клик → input → blur / Enter сохраняет. Используется для типов
цен, категорий, настроек агентов.

## Модалки

### Модалка подтверждения

```
┌── Cancel order #O-2026-0123 ────────┐
│                                      │
│  This will release reserved stock    │
│  and mark the order as cancelled.    │
│  This cannot be undone.              │
│                                      │
│           [Cancel]  [Confirm]        │
└──────────────────────────────────────┘
```

- Заголовок = глагол-действие + сущность (`Cancel order #…`).
- Тело ≤ 80 слов. Будьте конкретны о последствиях.
- Primary-действие справа; **деструктивный primary — красный**.
- Esc закрывает; Enter триггерит primary, кроме случая внутри textarea.
- **Не складывайте модалки.**

### Edit-в-модалке

Для быстрых правок (≤ 8 полей). Для ≥ 8 полей открывайте полную
страницу.

**Библиотека**: `js_plugins/fancybox2`.

## Уведомления

### Toast

Справа сверху, авто-закрытие через 5 с. Библиотека: `js_plugins/noty`.

### In-app колокольчик

Иконка-колокольчик в топ-баре → выпадающий список последних
уведомлений. Пункты ведут к соответствующей сущности.

### Баннер (на уровне страницы)

Сверху контентной области. Используйте для:

- «Лицензия истекает через 3 дня» (warning)
- «Ваша последняя синхронизация упала» (error)
- «Доступны новые release notes» (info)

## Графики

`js_plugins/jquery-highcharts-10.3.3`. Используйте:

- **Line** для трендов во времени.
- **Column** для сравнения по периоду.
- **Pie** с осторожностью — только для ≤ 5 долей, суммирующихся к 100 %.
- **Combo** (line + bar) для plan vs fact.

## Print-шаблоны

`protected/views/invoiceTemplate/`. Per-tenant, конфигурируется в
**Настройки → Шаблоны**.

## Mobile (web-админка) responsive

Админка — **desktop-first** (1280 × 800 минимум). Ниже:

- Сайдбар сворачивается до иконок.
- Таблицы переполняются по горизонтали со sticky-первой колонкой.
- Модалки становятся full-screen.

Phone-layout для web-админки нет — phone-пользователи используют
отдельное mobile-приложение.

## Когда добавлять новый компонент

Не добавляйте, если:

- Вы не можете собрать нужное из существующих частей.
- Та же потребность появляется на **3+** экранах.
- У вас есть одобрение от design lead.

Если да — добавьте PNG-вайрфрейм в `static/wireframes/` и строку в
этот каталог в том же PR.
