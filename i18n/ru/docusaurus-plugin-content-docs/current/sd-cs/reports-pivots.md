# Отчёты и сводные таблицы

В sd-cs есть два слоя отчётности:

| Модуль | Стиль |
|--------|-------|
| `report` | Одноэкранные отчёты — фильтры + таблица + экспорт в Excel |
| `pivot` | Сводные таблицы — перетаскивание измерений и метрик |

## Как они читают данные дилеров

Большинство отчётов проходит через паттерн **DealerReportService**:

1. Выбрать дилеров в области действия (страна, регион, все).
2. Для каждого дилера переключить соединение `dealer`.
3. Выполнить запрос на стороне дилера → отправить строки во временный сборщик.
4. Агрегировать в PHP после цикла.

Для очень больших сводных таблиц сервис записывает промежуточные строки
по каждому дилеру в стейджинговые таблицы `cs_*`, а затем агрегирует в MySQL.

## Кэширование

`redis_cache` (одно логическое соединение Redis) хранит:

- Сессии.
- Опциональные кэши результатов отчётов с ключом
  `report:<name>:<filters_hash>:<dealers_hash>`.

TTL в 5–15 минут типичны для отчётов HQ.

## Экспорт в Excel

Так же, как в sd-main — PHPExcel.

## Советы по производительности

- Всегда ограничивайте цикл по дилерам настраиваемой конкурентностью.
- Перенесите join-ы на сторону дилера; вытягивайте узкие строки.
- Кэшируйте агрессивно на стороне HQ; пользователей HQ устраивает
  устаревание в 1 минуту для метрик верхнего уровня.
