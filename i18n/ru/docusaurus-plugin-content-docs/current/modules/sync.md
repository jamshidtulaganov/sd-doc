# Модуль `sync`

:::info Минимальное покрытие
Это страница-заглушка — направление и логирование описаны, но инвентарь job'ов, расписания и tenant-specific overrides ещё не задокументированы. Вклад приветствуется через скилл `sd-docs-author`.
:::

Двусторонняя синхронизация данных, в первую очередь между SalesDoctor и
системой бухгалтерского учёта арендатора (обычно 1С). В основном задачно-ориентированный.

## Направления

| Тип | Где читать |
|------|---------------|
| Исходящий (SalesDoctor → 1С) | Задачи экспорта заказов в `integration` |
| Входящий (1С → SalesDoctor) | Задачи синхронизации каталога/цен/клиентов в `sync` |

## Логирование

Каждый запуск синхронизации логируется в `IntegrationLog` со статусом, payload
и любой ошибкой. UI `integration` читает из этой таблицы.
