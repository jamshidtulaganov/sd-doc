# Старт работы (фронтенд)

Эта страница — фронтенд-специфичный спутник общего раздела
[Локальная настройка](../project/local-setup.md). Сначала прочтите его; здесь
заполняются те части, которые нужны фронтенд-разработчику и которые общая
страница опускает (ответ на «Hot reload?» там — просто «нет», читайте дальше).

## Что вам реально нужно

- **Docker Desktop** — поднимает весь backend (web, MySQL, Redis).
  Это единственное обязательное требование, чтобы редактировать
  `protected/views/*.php`, `js/*.js`, `js_plugins/*`, `css/*`.
- **Node 18+** — нужен только если вы будете трогать:
  - сайт документации (`sd-docs`) — `npm install && npm run start`
  - Angular-«остров» (`ng-modules/gps`, `ng-modules/neakb`) —
    `npm install && npm run build`. См.
    [ng-modules](./ng-modules.md).
- **Современный Chromium** — dev-воркфлоу опирается на DevTools.
  Firefox работает, но примеры команды используют Chrome.

Локальный PHP **не нужен**. Весь PHP крутится в контейнере.

## Поднимаем

```bash
git clone git@github.com:salesdoctor/sd-main.git
cd sd-main
cp protected/config/main_sample.php protected/config/main_local.php
docker compose up -d --build
```

Web-приложение по адресу **http://localhost:8080** (согласно
[Локальная настройка](../project/local-setup.md)). Логиньтесь под
`admin / admin`.

## Откуда реально берётся frontend-код

Build-пайплайна для фронтенда нет (см.
[Пайплайн ассетов](./assets-pipeline.md)). Файлы отдаются как есть.
Ментальная модель:

| Вы редактируете | Браузер видит |
|-----------------|---------------|
| `protected/views/<ctl>/<action>.php` | Перерендер при следующем запросе |
| `js/<file>.js`, `js_plugins/<...>.js` | Подхватывается при перезагрузке браузера |
| `css/<file>.css` | Подхватывается при перезагрузке браузера |
| `ng-modules/<feature>/src/...` | **Только после** того, как `npm run build` пересоберёт `dist/` |
| Что угодно внутри `assets/<hash>/` | Не редактируйте — авто-сгенерированные копии |

Yii-овский `CAssetManager` копирует всё, что зарегистрировано как «asset
bundle», в `assets/<hash>/` и отдаёт хешированный URL. Когда вы редактируете
файл, попадающий в такой бандл, хеш меняется и браузер автоматически тянет
новую копию.

Для файлов, зарегистрированных напрямую через `clientScript->registerScriptFile`
(большинство `js/`, `js_plugins/`), URL стабильный и браузер может закешировать
старую копию. Две защиты:

1. **DevTools → Network → Disable cache**, пока DevTools открыт.
   Это стандартный воркфлоу фронтенд-разработки.
2. **Добавьте version-query** во view, чтобы форсировать hard-refresh
   на релизе:

   ```php
   Yii::app()->clientScript
     ->registerScriptFile('/js/orders.js?v=' . VERSION);
   ```

## Настройка браузерного DevTools

- **Disable cache** (вкладка Network) — всегда включён, пока открыт
  DevTools.
- **Preserve log** (Console + Network) — у приложения есть флоу с
  full-page-redirect, где иначе логи стирались бы.
- **Source maps** — их нет. Большинство файлов и так не минифицированы;
  отлаживайте их как есть.

## Smoke-тест чек-лист (первая сессия)

Перед тем как писать код, прокликайте это. Если что-то ломается, это
проблема настройки, а не вашего кода.

- [ ] Залогиньтесь как `admin / admin` на http://localhost:8080.
- [ ] Откройте одну **list-страницу** (например, заказы) — проверьте,
      что фильтры рендерятся и таблица отрисовывается.
- [ ] Откройте одну **form-страницу** (создание / редактирование) —
      проверьте, что Chosen-дропдауны и date pickers работают.
- [ ] Откройте один **модальный** флоу — проверьте, что fancybox-модалки
      открываются и закрываются.
- [ ] Откройте страницу **GPS-карты** (использует Angular-«остров»
      `ng-modules/gps`) — проверьте, что тайлы карты загружаются.
- [ ] Переключите локаль (топ-бар) между RU и EN — проверьте, что строки
      меняются.
- [ ] DevTools → Network — проверьте, что нет 404 на `js/`, `js_plugins/`,
      `css/`, или `assets/<hash>/`-URL.
- [ ] DevTools → Console — отметьте все uncaught-ошибки. Часть legacy-шума
      ожидаема; считайте это базовой линией, которую нельзя ухудшать.

## Hot reload?

Его нет. Общая страница [Локальная настройка](../project/local-setup.md) уже
говорит об этом. Frontend-воркфлоу такой:

- Отредактируйте файл в `protected/views/`, `js/`, или `css/`.
- Переключитесь в браузер.
- **Hard refresh**: `Cmd-Shift-R` (mac) / `Ctrl-Shift-R` (win/linux).
- В DevTools → Network должна стоять галка **Disable cache**, иначе
  hard-refresh не подхватит изменения в `js_plugins/` стабильно.

Для Angular-«островов» (`ng-modules/gps`, `ng-modules/neakb`) дефолтный
флоу сегодня — собрать-затем-обновить:

```bash
cd ng-modules/gps
npm run build
# затем hard-refresh браузера
```

Live-`ng serve` против живого Yii-бэкенда теоретически возможен, но сегодня
не задокументирован — см. открытые вопросы ниже.

## Tail-нем рантайм-лог

Большинство падений «страница пустая» или «модалка не сабмитится» —
**серверные ошибки**, которые фронтенд просто показывает пустым экраном.
Tail-ните лог:

```bash
docker compose exec web tail -f protected/runtime/application.log
```

Согласно [Соглашениям](../project/conventions.md), держите этот терминал
открытым всегда, когда отлаживаете AJAX-флоу.

## Частые подводные камни

- **`assets/` забивает диск** — периодически `rm -rf assets/<hash>` для
  устаревших опубликованных бандлов.
- **Права на `runtime/` и `assets/`** — должны быть доступны для записи
  пользователю `www-data` в контейнере; bind-mount с хоста может это сломать.
- **PHP-warning'и заваливают лог** — многие файлы из эпохи PHP 7.3.
  Оставайтесь на 7.3, если не уверены, что делаете.
- **Старые кеши в `protected/runtime/cache/`** — редко, но если view-partial
  кажется замороженным даже после редактирования, очистите эту папку.

  реальный символ может отличаться от репо к репо.
