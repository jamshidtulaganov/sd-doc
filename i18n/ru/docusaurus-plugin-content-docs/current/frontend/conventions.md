# Соглашения фронтенда

Эта страница покрывает соглашения, специфичные для слоя фронтенда. Для
PHP / DB / Git соглашений см.
[Проектные соглашения](../project/conventions.md). Для *анатомии* стека см.
[Обзор фронтенда](./overview.md).

## Где что лежит

| Если вы добавляете… | Положите под… |
|---------------------|---------------|
| JS-файл для конкретной страницы | `js/<area>.js` (first-party) |
| Стороннюю библиотеку | `js_plugins/<name>/` |
| Стайлшит для конкретной страницы | `css/<area>.css` |
| View-partial, переиспользуемый между страницами | `protected/views/partial/` |
| Переиспользуемый partial для модалки | `protected/views/modals/` |
| Интерактивный «остров» (Angular) | `ng-modules/<feature>/` |
| Vue-эксперимент | `protected/views/vue/` (с осторожностью) |

В `js/` уже примерно ~120 first-party файлов (согласно
[Структура проекта](../project/structure.md)), а `js_plugins/` — это
вендорный дом для third-party (Bootstrap 3, Chosen, fancybox, jQuery UI,
Highcharts, DataTables FixedColumns, noty — см.
[JS-плагины](./js-plugins.md)).

## Когда что использовать

Фронтенд намеренно микс. Правило выбора, по порядку:

1. **Обычная Yii-view + jQuery-сниппет** — дефолт. Используйте для
   list-страниц, форм, модалок и всего, что укладывается в существующий
   жизненный цикл страницы (см. [Обзор фронтенда](./overview.md#page-lifecycle)).
2. **Yii-view + first-party `js/<area>.js`** — когда страничный JS
   захламляет view, вынесите его в файл под `js/` и зарегистрируйте через
   `clientScript->registerScriptFile`.
3. **Существующая библиотека `js_plugins/`** — для известной нужды:
   график → Highcharts, поиск-селект → Chosen, таблица → DataTables,
   модалка → fancybox2, toast → noty.
4. **Новая библиотека `js_plugins/`** — только когда ни одна существующая
   не покрывает потребность *и* нет командного консенсуса написать своё.
   Каждое добавление перманентно (см.
   [JS-плагины · Добавление плагина](./js-plugins.md#adding-a-plugin)).
5. **Angular-«остров» под `ng-modules/`** — только для существенного
   интерактивного UI (live-карта, drag-drop канбан, сложная многошаговая
   форма). У каждого «острова» свой build, своё роутинг, своя стоимость
   поддержки. Перед добавлением нового — обсудите
   ([Обзор фронтенда](./overview.md#when-to-add-an-angular-island)).
6. **Vue-страница** — не добавляйте новых. Несколько существующих
   экспериментов в `protected/views/vue/` оставлены; новые SPA идут в
   `ng-modules/`, если они вообще должны существовать.

Если не уверены, какой уровень подходит — берите более низкий. Перейти от
jQuery-сниппета к Angular-«острову» редко дороже, чем обратно.

## Нейминг

### JS-файлы в `js/`

- **Page-specific** — называйте по контроллеру/области: `orders.js`,
  `clients.js`, `invoice.js`. Только нижний регистр.
- **Переиспользуемые утилиты** — называйте по предмету: `date-utils.js`,
  `money.js`. Избегайте `helpers.js` / `common.js` — они есть в каждом
  legacy-кодбейзе и никто не знает, что внутри.
- **Новая папка под `js/`** — только когда фича шипится в ≥ 3 файлах.
  Иначе оставляйте плоской.

### JS-файлы в `js_plugins/`

- **Одна папка на библиотеку** — `<name>/` совпадает с upstream-именем
  (`chosen/`, `fancybox2/`, `jquery-highcharts-10.3.3/`).
- **Закрепляйте версию в имени папки**, когда плагин когда-нибудь могут
  обновить рядом со старой версией (папка Highcharts — прецедент).

### CSS-файлы

- **Page-specific** — `css/<area>.css`, зеркаля JS-нейминг.
- **Site-wide** — минимально; большая часть стилизации сегодня едет на
  Bootstrap 3-классах.

### Yii views

Согласно [Соглашениям](../project/conventions.md):

- **Папка на контроллер** — `views/<controller>/`.
- **Один файл на экшен** — `views/<controller>/<action>.php`,
  нижний регистр.
- **Переиспользуемые partial'ы** — `views/partial/<name>.php`, включаются
  через `$this->renderPartial('partial.<name>', [...])`.

## i18n во views

Оборачивайте каждую видимую строку:

```php
<?= Yii::t('orders', 'Create order') ?>
```

- **Категория** — первый аргумент (`'orders'`, `'common'`, …). Совпадайте
  с модулем / областью, в которой находитесь — не сваливайте всё в
  `'common'`.
- **Source-string** — второй аргумент. Держите исходный язык одинаковым
  внутри категории (существующие каталоги мешают RU и EN исходники; при
  добавлении следуйте окружающему файлу).
- Каталоги живут в `protected/messages/<locale>/<category>.php`.
  Активные локали: **`ru` (по умолчанию)**, `en`, `uz`, `tr`, `fa`
  (согласно [Технологическому стеку](../architecture/tech-stack.md)).
- Добавлять новый ключ только в `ru` приемлемо для RU-first фичи, но
  планируйте EN + UZ внутри того же релиза. Чек-лист
  [стартового тикета онбординга](../team/onboarding.md#week-1) включает
  задачи «дозаполните недостающие переводы» именно поэтому.

## Вызов API из фронтенда

Согласно [Подводным камням первого месяца](../team/onboarding.md#pitfalls-to-avoid-in-your-first-month):

- **Не** добавляйте новые эндпоинты в `api` (v1) или `api2`.
- **Мобильные** флоу → `api3` ([API v3 — мобильный](../api/api-v3-mobile.md)).
- **Online / web** флоу → `api4` ([API v4 — online](../api/api-v4-online.md)).

Для Angular-«островов» см.
[ng-modules · Связь с PHP](./ng-modules.md#communication-with-php) —
контекст (тенант, пользователь и т.д.) идёт из Yii-view в Angular-бандл
через `data-*`-атрибуты на host-элементе.

## Что трогать нельзя

| Путь | Почему |
|------|--------|
| `framework/` | Вендорный Yii 1.x. Патчить только в крайнем случае и только через процесс команды. |
| `assets/<hash>/` | Авто-генерируемое. Правки перезатрутся, директория может быть удалена в любой момент. |
| `*.obsolete` | Удалённый код, оставленный для археологии. |
| `index2.php`, `a.php` | Legacy-эндпоинты / разовые скрипты. |
| `vendors/`, `protected/extensions/` | Закреплённые вендорные библиотеки. |

## Кандидаты на удаление (не расширяйте, планируйте вывод)

Из [JS-плагины · Кандидаты на удаление](./js-plugins.md#removal-candidates):

- **`bootstrap/`** — со временем заменить на utility-CSS.
- **`json2/`** — современным браузерам не нужен.
- **`fancybox/` (v1)** — оставить только `fancybox2`.

Если трогаете один из них — предпочитайте современную замену, если только
объём правки не реально bug-fix-in-place.

## Открытые вопросы / TODO

- **`js/`-кладбище** — есть ли окончательный список того, какие `js/*.js`
  файлы реально подключены каким-то контроллером через
  `registerScriptFile`, а какие заброшены? Пока такого списка нет —
  относитесь к незнакомым `js/`-файлам с подозрением и `grep`-айте
  ссылки до того, как считать их живыми.
- **Источник иконографии** — Bootstrap 3-глифы? кастомный спрайт?
  оба? См. [JS-плагины](./js-plugins.md#currently-used) для подсказок.
- **Site-wide globals**, экспортируемые существующим JS (например,
  `Distr` и другие) — должны быть перечислены здесь, как только
  подтвердятся.
