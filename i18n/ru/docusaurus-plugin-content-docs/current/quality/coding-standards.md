# Стандарты кода

## Стиль

- PSR-12, отступ четыре пробела.
- Одинарные кавычки для строк, если нет интерполяции.
- Всегда docblock для публичных методов.
- Не пишите `declare(strict_types=1)` в legacy-файлах.
- Предпочитайте ранние return вместо вложенных if.

## Правила pull request

- Одно логическое изменение на PR.
- Миграции — отдельным коммитом.
- Описание включает:
  - **Контекст** (почему)
  - **Что изменилось**
  - **Как проверить**
  - **План отката**

## Чего нельзя

- Не добавляйте новые файлы `*.obsolete`. Удаляйте мёртвый код, если
  уверены; сохраняйте через git history, если не уверены.
- Не расширяйте API v1 / v2. Добавляйте в v3 или v4.
- Не вызывайте кеш напрямую (`Yii::app()->cache`). Используйте `ScopedCache`.
- Не пишите SQL, таргетящий хардкод-имя БД.
