# Соглашения по данным

## Идентификаторы

- **Более новые таблицы** используют `ID INT AUTO_INCREMENT`.
- **Старые таблицы** используют `<ENTITY>_ID VARCHAR(32)`, содержащий
  синтетическую строку (часто `<filial>_<seq>` или UUID-подобный
  формат).
- **Внешние ID** живут в колонке `XML_ID`. Используется для round-trip
  с 1C, Didox, Faktura.uz.

## Регистр

Заглавные (`ORDER_ID`, `CLIENT_ID`) — legacy-дефолт; сохраняйте его на
таблицах, которые уже его используют. Новые таблицы могут использовать
snake_case в нижнем регистре, но будьте консистентны.

## Soft-delete

Почти везде `ACTIVE = 'Y' | 'N'`. Жёсткий `DELETE` зарезервирован для
admin-инструментов.

## Таймстемпы

- `CREATE_AT`, `UPDATE_AT` — `DATETIME`.
- `TIMESTAMP_X` — общий маркер last-modified, используется для
  sync-дельт.

## Кодировки

Дефолтная кодировка — `utf8` (3-байтовая). Некоторые более новые таблицы
используют `utf8mb4`. Будьте явны на `CREATE TABLE` для emoji-safe
текстовых колонок.
