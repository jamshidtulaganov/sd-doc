# Обзор безопасности

| Аспект | Решение |
|---------|-----------|
| Аутентификация | Учётные данные в БД, MD5 (legacy) — см. примечания |
| Сессии | Redis db0, `keyPrefix` = `HTTP_HOST` |
| Авторизация | `DbAuthManager` поверх `authitem` + кеш |
| Изоляция тенантов | Поддомен → БД; кеш с префиксом по тенанту + филиалу |
| Транспорт | TLS на nginx |
| Данные на диске | MySQL + файловая система; бэкапы шифруются |
| Секреты | `main_local.php` (не коммитится); креды 1C / Didox / Faktura.uz — в настройках тенанта |
| Аудит | `IntegrationLog`, `OrderStatusHistory`, таблицы `audit_*` |
| Rate limiting | Throttling логина по IP / по пользователю |

## Известные слабости (имейте в виду)

- **MD5-пароли** — историческое наследие; со временем нужно мигрировать на bcrypt.
  `LoginController` уже поддерживает оба варианта в переходный период.
- **Yii 1 CSRF** — включён по умолчанию, но несколько legacy-эндпоинтов POST
  обходят его. Аудитируйте перед добавлением новых.
- **PHP 7.3** — без upstream-поддержки безопасности. Компенсируется WAF
  и регулярными OS-обновлениями.
